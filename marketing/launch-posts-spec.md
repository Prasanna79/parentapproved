# ParentApproved.tv — Launch Posts Spec

## Goal
Announce the beta launch of ParentApproved.tv across Twitter/X, LinkedIn, and Reddit. Each post tells the "why I built this" story, signals beta honestly, and invites feedback. Voice: founder/parent, not brand.

---

## SEO & Discovery Keywords

**Primary (weave naturally, don't stuff):**
- kid-safe video player
- YouTube alternative for kids
- Android TV parental controls
- no algorithm kids TV
- open source parental control app

**Secondary / long-tail:**
- YouTube Kids alternative Android TV
- parent-approved videos only
- curated kids content
- screen time without guilt
- no ads kids TV app

**Hashtags (Twitter/LinkedIn only):**
- #AndroidTV #OpenSource #ParentalControls #ScreenTime #KidSafe #IndieDevs
- Avoid: #MomLife #ParentingHack (too generic, low signal)

---

## Platform Strategy

### Twitter/X
- **Format:** Thread (1 hook + 4-5 tweets + CTA)
- **Tone:** Founder talking to other parents and indie devs. Casual, direct. Light humor okay.
- **Audience:** Tech-adjacent parents, indie dev community, Android enthusiasts
- **Goal:** Retweets from indie dev / open source crowd, clicks to site
- **Key angle:** "I built this for my own kids" + open source + no algorithm
- **CTA:** Link to parentapproved.tv, invite feedback via DM or GitHub

### LinkedIn
- **Format:** Single post, ~200 words. No thread, no carousel.
- **Tone:** Professional-personal. "Here's what I built and why." Not a product launch announcement — a personal story with a link.
- **Audience:** Working parents, product/eng people, indie builders
- **Goal:** Engagement from the "building in public" crowd, signal to working-parent network
- **Key angle:** The problem (algorithm rabbit holes) → personal solution → now sharing it
- **CTA:** Link to site, "would love your feedback"

### Reddit

#### r/AndroidTV
- **Format:** Text post, 150-200 words. Technical lean.
- **Tone:** Fellow Android TV tinkerer sharing a project. Match the sub's matter-of-fact energy.
- **Audience:** People who own Android TV devices and tinker with them
- **Key angle:** What it does technically (sideload APK, no API key, ExoPlayer, manage from phone). Open source.
- **CTA:** Link to GitHub + site. "Happy to answer questions."
- **Rules check:** Self-promo is allowed if you're transparent ("I built this"). Don't post as a company.

#### r/Parenting
- **Format:** Text post, ~200 words. Personal story.
- **Tone:** Parent first, developer second. Vulnerable, not salesy.
- **Audience:** Parents dealing with screen time decisions daily
- **Key angle:** "YouTube is a minefield. I got tired of my kid drifting into weird videos. So I built something where I pick every video." Beta, free, open source, no catch.
- **CTA:** Link to site. "Curious if other parents would find this useful."
- **Rules check:** r/Parenting allows personal projects if framed as sharing, not advertising. Lead with the problem, not the product.

#### r/opensource
- **Format:** Text post, 150 words. Project showcase.
- **Tone:** Developer sharing a project. Technical details welcome.
- **Audience:** Open source enthusiasts
- **Key angle:** Kotlin + Jetpack Compose + Cloudflare Workers. Local-first, no cloud accounts. Charityware.
- **CTA:** GitHub link. "PRs welcome, feedback appreciated."

---

## Story Arc (Consistent Across All Platforms)

1. **The problem:** My kid starts on dinosaur videos and ends up watching... something I wouldn't choose. YouTube Kids still has an algorithm. I wanted zero algorithm.
2. **What I did:** Built an Android TV app where you paste playlist URLs and kids see only those videos. No search, no recommendations, no ads.
3. **How it works (brief):** Manage from your phone, videos play on the TV. Free, open source, no account needed.
4. **Honest about stage:** This is a beta. Some things are rough. I'm actively working on it.
5. **The ask:** Would love feedback from parents / Android TV users / open source folks.

---

## Anti-Patterns (Per Style Guide)

- No fearmongering ("PROTECT your kids from DANGEROUS content!")
- No corporate speak ("We're excited to announce our innovative solution")
- No guilt ("Are YOU monitoring what your children watch?")
- No fake urgency ("Limited beta spots!")
- No over-promising ("Never worry about screen time again!")
- No "we" — it's "I" (founder voice)
- YouTube always as descriptive reference, not identity

---

## Measurement

- **Twitter:** Impressions, retweets, link clicks, replies
- **LinkedIn:** Reactions, comments, profile views
- **Reddit:** Upvotes, comment count, traffic to site (check analytics)
- **All:** GitHub stars in the 48h after posting

---

## Timing

- Post Reddit threads first (slower burn, longer shelf life)
- Twitter thread same day
- LinkedIn next morning (weekday, work hours)
- Don't post all Reddit subs on the same day — space 1-2 days apart to avoid looking spammy
